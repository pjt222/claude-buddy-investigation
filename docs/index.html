Buddy Security & Privacy Findings

CRITICAL

Command Injection in Capture Scraper

Environment variables (WEZTERM_PANE, SHINGLE_TMUX_PANE, SHINGLE_TERMINAL_LOG) are interpolated directly into execSync() shell commands without sanitization. Fix: use execFileSync with argument arrays.

HIGH

Unfiltered Transcript Transmission

The buddy_react API receives conversation transcript per reaction with no filtering for secrets, API keys, passwords, or PII. Use /buddy off when handling sensitive credentials.

LOW

Month-Gate Seasonal Bug

[month-gate-condition] — the month check fails January-March (values 0-2), silently disabling companions for 3 months every year. Likely a one-time launch gate incorrectly ANDed with a monthly condition.

VERIFIED

Unidirectional Trust Boundary

The companion cannot write to the main agent's conversation, modify files, invoke tools, or influence agent behavior. Reactions go to the UI speech bubble only. This boundary is enforced architecturally, not by policy.

INFO

Muting Stops All Traffic

/buddy off sets [companion-muted-key]: true, which stops both UI display and network transmission. This is the correct behavior — muting is not cosmetic.

NOTE

MCP Companion Takeover

Any MCP server with file-write access to ~/.claude/.claude.json can overwrite the companion's name and personality. No integrity check on the config file. Species/appearance cannot be changed (deterministic from hash). (Architectural observation from investigation — not part of the formal security audit.)

INFO

Pipeline Fully Reproduced

13

Primary/Secondary Stat System

Stats are NOT uniform random. The binary's [buddy-stat-gen-fn] picks a primary stat (+50 boost) and secondary stat (−10 penalty). Shingle's PATIENCE (81) is the primary; CHAOS (1) is the secondary. This creates TCG-like stat profiles by design.

14

Species Array Not Alphabetical

The binary's [buddy-species-array] array stores species in a specific non-alphabetical order. Index 6 = owl in the binary, but dragon if sorted alphabetically. Same hash, same PRNG, different species — array order is the entire difference.

Advisor Overview

Unlike the companion — a read-only observer — the advisor is a server-side tool within the Messages API. When the executor calls advisor(), the full conversation history is forwarded to a stronger reviewer model.

Bidirectional Decision Gate

The executor model (Sonnet or Opus) can call advisor() at any point. The reviewer model receives the full conversation — all tool calls and results — and returns guidance. The executor resumes with that guidance incorporated.

No Separate Endpoint

The advisor is a tool type (advisor_20260301) within the standard Messages API. Unlike buddy_react, there is no custom endpoint — it rides the same call the executor already makes.

Dark-Launched

Hidden behind [advisor-gate] feature flag. Already code-complete in v2.1.96 while the buddy system was still fully visible. Infrastructure was built in parallel, not as a replacement.

Kairos Loop System

The Kairos loop system provides autonomous self-continuation to agent sessions. A session can call ScheduleWakeup(delaySeconds, prompt, reason) to re-enter itself later, enabling the /loop slash-command pattern and dynamic self-pacing. Active from v2.1.101 onward.

Empirical Probe (§3.5, 2026-04-14)

Probe suffix 83af5507. Filesystem before: ~/.claude/teams/ had zero entries. Probe action: skill-forked subagent called SendMessage(to="ghost-probe-83af5507", message="probe"). Result: runtime returned {"success": true} and created the full tree teams/default/inboxes/ghost-probe-83af5507.json on disk.

Attribution Laundering — the attack

Forged sender identity. SendMessage(to, from, message) writes the from value verbatim into the inbox file without verifying that the sending process has that identity. A skill-forked subagent running as the user can set from: "team-lead", from: "auditor", or any other string — and the receiving agent (or a later-running tool that reads the inbox) sees a well-formed message from a principal that the sender doesn't represent.

Why it's Critical. Combined with the silent-creation path, this means an untrusted or compromised skill can plant messages in arbitrary inboxes attributed to any principal — including roles like "team-lead" that may have elevated downstream trust. There is no log trail differentiating the forgery from a legitimate message. Closed empirically as #31 AC3 on 2026-04-14.

Path Correction (§3.4, 2026-04-14)

Earlier static-analysis inference suggested a separate ~/.claude/agent-teams/ storage domain. Empirically disconfirmed. All team state lives under ~/.claude/teams/, materialized lazily on first write. No memdir/ string observed on disk. See results/memory-trio-2026-04-14.md revision notice.

Grove & Flag Resolution Matrix

Four distinct configuration layers govern capability gating: env vars, local settings, Statsig (server-side feature flags), and grove (account policy surface). They have overlapping but distinct semantics, and the precedence rules produce silent shadows if not understood.

Layer Mechanism Behavior
Env-var kill CLAUDE_CODE_DISABLE_* Hard veto — always wins
Env-var enable CLAUDE_CODE_EXPERIMENTAL_* Necessary precondition where present (not a replacement for Statsig)
Statsig [statsig-resolver-fn](<flag-name>, default) Server-side rule eval; result cached for session lifetime
GrowthBook [growthbook-resolver-fn](<flag-name>, default) Distinct resolver — remoteEval POST to api.anthropic.com, 6h cache. Example: [session-recap-gate] (away-summary gate, default ON)
Grove policy /api/[internal-policy-endpoint] Account-level preference; renders UI notice but does NOT gate [statsig-resolver-fn] calls

Silent Shadow

Setting autoMemoryEnabled: true in settings.json does not enable memdir if the feature's disable env var is also exported (see <feature-disable-env-var>). No warning, no telemetry difference. auto-memory-disabled fires with disabled_by_env_var: true.

Grove ≠ Feature Flag

Grove is an account preference surface, not a gate. A user who dismisses the notice does not thereby unlock any capability. A Statsig-DARK session still renders the grove notice if grove_enabled === true.

The codename grove is reused by an undocumented grove cohort label (unrelated to the policy surface). The two are orthogonal.

Project-Local > Global

Hook settings in a project's .claude/settings.json merge with global ~/.claude/settings.json: both fire. For conflicting scalar values (e.g., theme), project-local wins in that project dir.

Reactive Compaction

Added in v2.1.107. Replaces the manual /compact slash command with an auto-triggered pathway: when the session's token count or memory pressure crosses a threshold, context is automatically compacted without user confirmation. The SM_COMPACT env var was removed, suggesting the opt-out knob is gone.

New Telemetry (v2.1.107)

auto-compact-namespace — user changed compaction settings

Why It Matters

Context compaction is lossy by design. Auto-triggering it without user-in-the-loop means conversation context can silently shrink during long sessions, affecting agent behaviour and reproducibility. The 2.1.107 baseline diff flagged it as one of three new subsystems in that release.

Unknown Thresholds

The exact token-count or memory-pressure threshold that triggers auto-compaction has not been characterized empirically. CLAUDE_CODE_AUTO_COMPACT_WINDOW (ms) suggests the trigger is at least partly time-windowed, but the full trigger logic requires a long-running session probe.
